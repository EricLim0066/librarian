Key Research Discoveries

1. Environmental Stress Test

Finding: Modern browser engines (e.g., V8) almost match Native performance under optimal conditions. However, Native apps remain more stable (under 2s) during network degradation because they rely on pre-installed local assets.

2. Responsiveness & Synchronization

Finding: Native apps utilize direct OS communication channels, reducing UI lag. Web apps must route requests through the browser's JavaScript engine, adding processing overhead at every interaction point.

3. Real-Time Synchronization

Finding: Native apps leverage low-level TCP/UDP sockets for near-instant synchronization. Web apps are bound by WebSocket constraints and browser-level polling, leading to higher perceived latency.

4. Framework Overhead (The Canva Case)

Finding: Contrary to intuition, the Canva Web version launched faster than the Desktop Native version.

Analysis: The Native app is an "Electron Hybrid" that must cold-boot a redundant Chromium browser shell, causing higher memory usage compared to an optimized browser tab.

5. Scope Study of WhatsApp

Mobile Web Limitations: Authentication loops, poor UI scaling, and JavaScript sandbox limitations for E2EE decryption lead to a degraded experience compared to the multi-threaded, hardware-accelerated Native app.
